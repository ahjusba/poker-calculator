## Key Principles

1. **Fast**: Transaction-based isolation = milliseconds per test
2. **Isolated**: Each test is independent (no shared state)
3. **Realistic**: Tests use actual database (not mocks)
4. **Safe**: Separate test database (never touches prod/dev)
5. **Comprehensive**: Unit + integration coverage
6. **Maintainable**: Clear structure, helper functions, fixtures
